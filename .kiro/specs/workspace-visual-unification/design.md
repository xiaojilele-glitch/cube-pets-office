# Workspace Visual Unification - 设计文档

## 概述

本设计通过“设计 token + 共享页壳 + 重点页面治理”的方式，统一系统视觉语言，同时避免让不同功能页失去各自的工具属性。

## 设计原则

1. 一致性优先于个性化拼贴
2. 统一主题，但不抹平功能差异
3. 样式优先 token 化，避免继续散落内联
4. 视觉统一服务理解，不服务装饰

## 统一方向

### 1. 主题基底

- 延续办公室与工作流面板已有的暖色工作台气质
- 避免冷色工具页与白板风页继续并存

### 2. 共享规则

- 页面背景层级
- 卡片容器
- 标题与说明文本层级
- 按钮与状态标签映射

### 3. 优先治理页面

1. `client/src/pages/lineage/LineagePage.tsx`
2. `更多` 入口中的低频管理页
3. 仍保留的独立指挥中心兼容页

## 实现策略

### 1. Token 层

建议优先建立：

- 页面背景色组
- 卡片背景与描边
- 主次按钮
- 风险状态色
- 阴影与圆角

### 2. 页壳层

建议新增统一工作台页壳，统一：

- 顶部标题区
- 过滤区 / 工具区
- 主内容容器
- 空态 / 错误态占位

### 3. 页面治理顺序

- 先治理低频页面，风险低、收益高
- 任务页等主路径页面放在布局稳定后再统一细节

## Worktree 并行建议

### 推荐 owner

- Worktree F: 主题 token、共享页壳、`LineagePage.tsx`、低频工具页样式

### 可以并行

- 可在导航骨架稳定后，与 `scene-agent-interaction` 并行做样式收尾
- 可与 `api-fallback-empty-states` 协同统一空态与错误态视觉

### 不建议并行

- 不建议在 `task-hub-convergence` 高频改版期重做 `TasksPage` 细节样式
- 不建议与导航 spec 同时重写首页壳层视觉

## 测试策略

- 关键页面视觉回归截图
- 主题 token 使用检查
- 移动端与桌面端可读性检查

## 交付顺序

1. Token 与共享页壳
2. `LineagePage` 治理
3. `更多` 页面治理
4. 主路径页面样式收尾
