# 发布稳定性护栏方案 v2 手工验证

## 1. 新人启动验证

1. 按 README 执行
2. 验证 3 步内可跑起前端与服务
3. 验证主要页面可访问

## 2. 构建验证

1. 运行 `npm run lint`
2. 运行 `npm run typecheck`
3. 运行 `npm run test`
4. 运行 `npm run build`

## 3. 关键链路验证

1. 发起任务
2. 观察 executor 正常执行
3. 触发 decision
4. 完成任务并查看结果

## 4. 错误恢复验证

1. 人为断开 websocket
2. 验证自动重连
3. 人为制造 executor 超时
4. 验证任务进入失败或可恢复状态，而不是无声卡死

## 5. 任务恢复验证

1. 执行中刷新页面或重启服务
2. 验证页面能重新 attach 当前任务或恢复足够上下文
