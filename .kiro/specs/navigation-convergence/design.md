# Navigation Convergence - 设计文档

## 概述

本设计把当前“8 个并列入口”的首页工具栏，收敛成 `办公室 / 任务 / 更多` 三段式主导航。重点不是删功能，而是重排主次关系。

## 设计原则

1. 主路径优先于功能完整陈列
2. 一级入口只服务“看态势”和“推进任务”
3. 低频能力收纳，但不隐藏到不可达
4. 旧路由不断链，迁移要可回退

## 目标信息架构

| 一级入口 | 角色                | 承载内容                               |
| -------- | ------------------- | -------------------------------------- |
| 办公室   | 默认首页 / 全局态势 | 3D 场景、公告板、Agent 状态、全局摘要  |
| 任务     | 主操作中心          | 命令输入、任务列表、执行进度、人工干预 |
| 更多     | 低频工具收纳        | 配置、权限、审计、血缘、帮助           |

## 路由策略

### 1. 保留现有主路由

- `/` 继续对应首页，但文案与导航明确为 `办公室`
- `/tasks` 作为主操作台
- `/command-center` 暂时保留，用于兼容重定向或过渡提示
- `/lineage` 等低频页面继续保留独立路由

### 2. 导航不等于路由数量

路由可以继续存在多个，但一级导航只暴露三个入口。低频页面从 `更多` 进入，不再直接参与首页主路径竞争。

## 组件改造范围

### 1. Toolbar / HoloDock

- 将主按钮收敛为三项
- `更多` 使用抽屉或菜单承载低频入口
- 原 `workflow`、`commandCenter` 等按钮不再直接出现在 dock 主层

### 2. Home 页面

- 首页继续保持 3D 场景中心
- 弱化“打开多个工具面板才能开始”的感觉
- 提供通往 `任务` 页的明确 CTA

### 3. App 路由

- 保持兼容路由
- 为未来 `/command-center -> /tasks` 的收口预留重定向方案

## 迁移策略

### 第一阶段

- 完成三段式导航骨架
- `更多` 收纳低频页面入口
- 旧路由继续存在

### 第二阶段

- 配合 `task-hub-convergence` 将 `command-center` 主路径收口到 `/tasks`
- 配合 `workflow-panel-decomposition` 移除首页对 `workflow` 的一级依赖

## Worktree 并行建议

### 推荐 owner

- Worktree A: `client/src/App.tsx`、`client/src/components/Toolbar.tsx`、`client/src/pages/Home.tsx`
- Worktree A: 新增 `MoreDrawer` 或等效导航收纳组件

### 可以并行

- 可与 `task-hub-convergence` 并行，但前提是双方约定好 `任务` 页最终仍挂在 `/tasks`
- 可与 `api-fallback-empty-states` 并行，冲突较低

### 不建议并行

- 不建议与 `scene-agent-interaction` 同时大改 `Home.tsx`
- 不建议在未冻结导航结构前启动 `workspace-visual-unification` 的首页壳层样式重做

## 测试策略

- 主导航渲染与 active 状态测试
- `更多` 抽屉入口测试
- 旧路由兼容性测试
- 桌面端与移动端快速回归

## 交付顺序

1. 信息架构映射定版
2. Toolbar 改成三段式
3. 接入 `更多` 抽屉
4. 路由兼容与回归
