# 任务操作系统首页重构方案 v1

## 目标

把首页固定为一个稳定的四区任务操作系统壳，而不是继续作为“3D 展示页 + 各种功能入口拼盘”。

## 适用范围

仅覆盖桌面首页 `/` 的主界面重构，包括：

- 顶部状态条
- 左侧场景区
- 中间任务主线区
- 右侧控制区
- 底部运行区

不覆盖：

- 任务详情页深层信息结构
- 回放页重写
- 移动端全量适配

## 首页必须固定成四区

### 左侧场景区

- 3D 办公室只作为状态氛围层
- 显示当前任务总状态
- 显示 agent 执行态或焦点态
- 不承载主流程按钮

### 中间任务主线区

- 必须成为第一视觉中心
- 展示 `plan -> execute -> review`
- 当前步骤必须高亮
- 每步必须可见状态、类型、摘要

### 右侧控制区

- 统一放输入、澄清、decision、取消/继续/重试等控制动作
- 不再把这些关键交互分散到其他页面

### 底部运行区

- 统一承接日志、artifact、executor 状态
- 不再散在多个 panel

## 体验要求

- 用户打开首页后能在 2 到 3 秒内理解“现在任务做到哪里”
- 中间任务主线区要明显强于左右两侧
- 右侧控制区必须在需要人工参与时立即可用
- 3D 场景不能压过任务主线
- 页面层级必须明显收敛，不再像多个工作台并列竞争

## 兼容要求

- 保持现有 mission / workflow / task store 业务真相源不变
- 保持澄清链路和任务创建链路可继续使用
- 不改后端契约

## 验收标准

- 首页可以独立承接 MVP 黄金路径
- 用户无需跳页即可完成“输入 -> 计划 -> 执行 -> 决策 -> 查看结果”
- 底部运行区承担运行证据展示
