# 任务操作系统首页重构方案 v1 手工验证

## 1. 首屏主次验证

1. 打开首页
2. 观察第一眼是否先看到中间任务主线区
3. 验证 3D 场景没有压过任务信息

## 2. 输入与澄清验证

1. 在首页右侧直接输入任务
2. 验证无需跳页即可提交
3. 若需要澄清，验证澄清问题直接出现在右侧控制区

## 3. 任务主线验证

1. 提交任务后观察中间区
2. 验证 `plan -> execute -> review` 清晰可见
3. 验证当前步骤高亮
4. 验证等待决策时中间区明确显示 waiting 状态

## 4. 控制动作验证

1. 在右侧控制区触发 approve / reject / modify
2. 验证动作不再隐藏在日志或其他详情 panel 中

## 5. 底部运行区验证

1. 打开底部运行区
2. 验证 logs、artifacts、runtime 三类内容都集中在这里
3. 验证没有关键运行证据散落到其他 panel
