# 回放与调试面收口方案 v1 手工验证

## 1. 回放入口验证

1. 完成一个任务
2. 从任务结果进入 `/replay/:missionId`
3. 验证回放页可正常加载

## 2. 回放内容验证

1. 验证可以看到计划
2. 验证可以看到关键步骤
3. 验证可以看到决策点与结果
4. 验证可以看到输出结果或关键证据

## 3. Debug 面验证

1. 进入 `/debug`
2. 验证 lineage / audit / permission / config 能在此归口
3. 验证普通主导航中不再高频暴露这些能力

## 4. 主流程干扰验证

1. 作为普通用户重新走一次首页主流程
2. 验证不会因为低频调试能力而分散注意力
