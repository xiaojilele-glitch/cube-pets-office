# Scene Agent Interaction - 需求文档

## 概述

当前 3D 办公室场景已经具备 Agent 选中与状态展示基础，但仍偏“观赏层”。用户看完后很难继续深入了解某个 Agent 当前在做什么、是否被阻塞、积累了哪些记忆，以及整个任务流目前推进到了场景中的哪一段。

本 spec 的目标是让 3D 场景从“可视化背景”升级为“可交互态势入口”。

## 设计目标

1. 点击 Agent 可展开完整上下文
2. 办公室墙面承载关键态势摘要
3. 任务推进能在场景中被一眼读懂
4. 无服务端时仍然保持可解释的演示体验

## 依赖关系

- 建议在 `navigation-convergence` 之后推进
- 与 `workflow-panel-decomposition` 协同承接 `org / memory / reports`

## 非目标

- 本 spec 不重做整个 3D 美术资源
- 本 spec 不改变任务页的主操作闭环
- 本 spec 不负责所有页面的主题统一

## 用户故事与验收标准

### 1. Agent 详情侧栏

#### 1.1 作为用户，我希望点击场景中的 Agent 后就能知道它是谁、现在在做什么、状态是否正常

- AC 1.1.1: 点击 Agent SHALL 打开详情侧栏
- AC 1.1.2: 侧栏 SHALL 展示角色、部门、当前任务、心跳状态、信誉分
- AC 1.1.3: 侧栏 SHALL 提供近期记忆、历史经验或报告摘要入口

### 2. 公告板

#### 2.1 作为用户，我希望进入办公室第一眼就能看到现在最重要的几件事

- AC 2.1.1: 办公室 SHALL 提供公告板或等效摘要面板
- AC 2.1.2: 公告板 SHALL 至少显示执行中任务数、阻塞 Agent 数、Token 或成本摘要
- AC 2.1.3: 公告板 SHALL 支持点击进入相关任务或页面

### 3. 场景流线

#### 3.1 作为用户，我希望任务推进在空间中可感知，而不是只在文本进度条里出现

- AC 3.1.1: 任务执行时 SHALL 在场景中显示阶段流线或等效动态路径
- AC 3.1.2: 场景流线 SHALL 有稳定的 stage-to-zone 映射
- AC 3.1.3: 无运行任务时 SHALL 不出现误导性的流线噪音

### 4. 演示与降级

#### 4.1 作为用户，我希望在前端预览或演示模式下也能理解场景在表达什么

- AC 4.1.1: 无服务端连接时 SHALL 显示演示模式解释文案
- AC 4.1.2: 缺失心跳、记忆或报告数据时 SHALL 显示明确空态说明
- AC 4.1.3: 场景交互 SHALL 不因缺少实时数据而崩溃

## 约束与风险

- `Home.tsx`、`Scene3D.tsx`、`PetWorkers.tsx`、`OfficeRoom.tsx` 容易与导航改造冲突
- 若没有统一的 stage-to-zone 映射，流线会变成一次性视觉特效而非稳定语义
- Agent 侧栏的数据源需要跨 `workflow-store`、`tasks-store`、`reputation-store`
