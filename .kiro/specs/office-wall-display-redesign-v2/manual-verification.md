# 办公室后墙三屏原生化重构手工验证（v2）

## 目标

确认桌面办公室首页后墙三屏从“网页浮层”升级为“统一设备”后，视觉主次、任务焦点、交互路径和现有业务链路保持正确。

## 验证环境

- 桌面浏览器打开办公室首页
- 使用默认桌面镜头进入首页
- 保持任务、sandbox、截图流可正常触发

## 核心观察项

### 1. 默认桌面镜头首屏可读性

1. 打开办公室首页，停留在默认桌面镜头。
2. 在 2 秒内观察是否能读出中间任务区的标题、状态和进度。
3. 确认左中右三块区域看起来像一台统一后墙设备，而不是三张独立网页卡片。

预期结果：

- 中间任务区为主视觉中心。
- 左右两区明显是辅助证据区。
- 后墙整体具备设备感和内嵌感。

### 2. 五类状态检查

分别准备或模拟以下状态：

- 无任务
- 等待中
- 运行中
- 失败
- 完成

每一类状态都检查：

1. 中间任务区结构是否稳定。
2. 左侧执行流是否保留合理状态表达。
3. 右侧浏览器区是否保留正确空态或最后有效画面。
4. 三块区域是否仍围绕同一任务焦点同步。

### 3. 左中右三块区域点击行为

1. 点击左侧执行流区域。
2. 点击中间任务主控区域。
3. 点击右侧浏览器回传区域。

预期结果：

- 左侧进入终端相关聚焦或详情路径。
- 中间进入任务详情或任务聚焦路径。
- 右侧进入截图或浏览器相关聚焦路径。
- 不再只有一个粗粒度全局 fullscreen 感受。

### 4. 窗口缩放与结构稳定性

1. 在桌面宽度下缓慢拖动浏览器窗口。
2. 观察后墙设备壳体、三分区比例和文本可读性是否稳定。
3. 观察后墙是否重新退化为“悬浮网页卡片”的观感。

预期结果：

- 三分区结构不塌陷、不漂移。
- 中间区始终强于左右两区。
- 设备感仍然成立。

### 5. 与底部 OfficeTaskCockpit 的主次关系

1. 保持底部 `OfficeTaskCockpit` 正常显示。
2. 观察后墙与底部驾驶舱是否发生严重视觉冲突。
3. 判断后墙是否承担“远距离主状态”，底部是否承担“近距离操作”。

预期结果：

- 两者可共存。
- 主次清晰。
- 若仍有冲突，应记录为后续视觉统一议题，而不是本轮阻塞项。

### 6. 现有路径回归

检查以下路径是否保持可用：

- `/tasks`
- 任务详情页
- 终端详情或终端放大路径
- 截图放大或浏览器预览路径

预期结果：

- 深链不回归。
- 数据来源不回归。
- 任务、终端、截图三条链路仍与场景联动。

本 v2 手工验证重点对照以下旧问题：

1. v1 仍然读起来像三张贴墙网页卡片。
2. 左侧执行流仍保留桌面终端窗口语义。
3. 中间任务区仍保留纸卡和 mounted 卡片语义。
