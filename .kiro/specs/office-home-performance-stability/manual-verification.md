# 办公室首页性能稳定性优化手工验证

## 1. 硬刷新验证

1. 打开桌面办公室首页。
2. 切换到 Chrome DevTools 的 Performance 面板。
3. 勾选 Screenshots，开始录制。
4. 对首页执行一次硬刷新。
5. 观察：
   - 首屏是否比改造前更快进入稳定状态
   - 是否仍能看到办公室房间主体、任务焦点和主要交互
   - 网络面板中是否不再在模块加载瞬间触发全量 GLTF 预热

## 2. 连续拖拽宽度验证

分别在 `1280 / 1440 / 1728` 宽度附近执行以下步骤：

1. 保持办公室首页处于桌面模式。
2. 用鼠标持续拖拽浏览器宽度约 5 秒。
3. 观察：
   - 页面是否仍保持可响应
   - 顶部按钮和驾驶舱是否不再出现明显掉帧
   - 3D 场景是否在拖拽期间切换到更轻量的表现
   - 停止拖拽后视觉是否恢复

## 3. 联动回归

1. 点击任务，确认右侧详情仍能同步切换。
2. 点击场景中的 Agent，确认选中与详情联动不回归。
3. 确认等待决策气泡、沙盒左右预览面板仍可正常工作。
4. 在高级模式下确认 heartbeat、workflow、任务列表仍能正常刷新。

## 4. 观察点

- 若拖拽仍有明显卡顿，优先检查：
  - 非关键 `Html` 是否仍然过多
  - `useFrame` 动画是否仍在 resizing 中运行过多
  - 是否还有组件直接消费连续 width 导致频繁重渲染
- 若首刷仍偏重，优先检查：
  - `OfficeRoom` 首屏仍渲染的 GLTF 数量
  - 是否还有模块层全量 `useGLTF.preload(...)`
  - 首页是否仍有重复的数据初始化链路
